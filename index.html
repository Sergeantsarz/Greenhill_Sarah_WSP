Web Standards Project

This website is dedicated to the absorption and process of solidifying and honing our skills with HTML5 and CSS3 in order to create content on the web that is visually appealing, semantically relevant, and 100% validated. Inspiration for the focus of our research and development will be garnered from an outside source. The site that we have been provided with for inspiration is called http://www.templatemonster.com/.

Main Focus

The Header: This component of a web page can be found all over the internet and contain a myriad of things like navigation, logos, information, advertisements, and more.

What I'd like to learn:

There are plenty of things that a programmer could choose from to focus on when it comes to the many different sorts of elements found in a typical web page. It's our task, during this month, to choose a component that we would most like to improve upon and practice using.

How Would I Make This

Step One: Use the < header > tag to start the section of content where all the header's contents will be located.

Step Two: Between the open and closing tags of the header, decide what is going to be contained within.

Step Three: Decide on creating a navigation menu. Use the < nav > tag to create a navigation within the previously created < header >.

Step Four: Use the < ul > (unordered list) tag inside the < nav > tags.

Step Five: Use the < li > tag within the newly created unordered list for as many items as the menu needs.

Step Six: Use further < ul > tags containing < li > elements within the navigation menu in order to create any needed submenus or drop-downs.

Five Areas of Sub-Focus

CTA

The first element that I would like to focus on is the CTA, or "Call to Action".

Definition

In web design, a CTA is a banner, button, or some type of graphic or text on a website meant to prompt a user to click it and continue down a conversion funnel. It is an essential part of inbound marketing as well as permission marketing in that it actively strives to convert a user into a lead and later into a customer. The main goal of a CTA is a click, or a scan in the case of a qr code, and its success can be measured via a conversion rate formula that calculates the number of clicks over the times the CTA was seen. - Wikipedia

How To

Most websites have some kind of CTA these days, especially if the site has anything to do with e-commerce or anything similar to that, but a "call to action" can be beneficial, leading the user of the site farther into the website itself.

Main Info / About Me

The second element of a web page that I would like to focus on is the main section of a web page where most people may put the most important information about themselves, their product(s), and/or services. The About Me/About Us/Main Info section is where the user of the site is going to get the biggest chunk of useful info and probably one of the first things they see.

Definition

The main section of information or "About Me" on a website is one of the most important and integral parts of a site, which goes unused or underplayed a lot of the time. Contained in this section are the answers to these key questions: Who am I and what can I do for you, the visitor?

How To

This area of the page will be contained within its own < section > element and possible a < div > or two, depending on the direction the main info section is being taken. The utilization of all the typical HTML5 elements will be present, including but not limited to: < p > tags, < a> tags, < img > tags, and more. The object is to grab the user's attention, keep it, and convey any pertinent information necessary in one spot near the top of the page.

Footers

An often overlooked and seemingly unimportant website component, the footer is something else I would like to focus on, because every part of your page should be as engaging as the last, including the footer.

Definition

A footer element typically contains: Authorship information, copyright information, contact information, a sitemap, back to top links, and/or related documents. - W3C

How To

HTML5 has a footer element that can be used to contain and display all these sorts of things, and most of the time, the footer is the last (and least attention-getting) element on the web page that a person sees. One of the most important things that I would like to get into the habit of doing is making sure that within my footer, I include a site map if the site I have created is spanned across multiple pages.

Contact / Forms

Forms in a web page are something that's almost surely extremely useful to the developer, they can have a variety of purposes in a page, and therefore create a bit of diversity in what you can do with them. I have chosen this component because I believe that they're under-used in general, and by me, so I want to get more into the habit of making sure to utilize form fields.

Definition

HTML Forms are one of the main points of interaction between a user and a web site or application. They allow users to send data to the web site. Most of the time that data is sent to the web server, but the web page can also intercept it to use it on its own. An HTML Form is made of one or more widgets. Those widgets can be text fields (single line or multiline), select boxes, buttons, checkboxes, or radio buttons. Most of the time, those widgets are paired with a label that describes their purpose. - MDN

How To

Creating form fields is pretty easy in this era of the web. Using the corresponding elements like < textarea > and < label > within the < form > tags are quick and painless ways to give the user of your site a means to contact you. You start with the < form > element and add whatever is needed within (like input fields, check boxes, labels, and buttons) depending on what the form is being used for.

Recent Articles

A lot of personal pages have blogs or articles that are posted for the consumption and enjoyment of those that visit the site and typically, the most recently posted ones are displayed to the user in a section of the front page of the site so that they can easily access new content. I would like to focus on this, designing and coding it in a way that is eye-catching and easy to use.

Definition

A blog (a truncation of the expression weblog) is a discussion or informational site published on the World Wide Web and consisting of discrete entries ("posts") typically displayed in reverse chronological order (the most recent post appears first). - Wikipedia

How To

Each image for its corresponding post will need to be located in an images folder within the component folder so that the articles have a visual representation of the most recently added posts. Aside from this, the component will need to be nested within a < section > element, be floated so that the posts sit along side one another properly, and contain enough information so that the user can quickly determine which post they would like to read in full with just a small excerpt of the whole article.
